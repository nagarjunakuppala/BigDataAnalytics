Clustering¶

The goal is to determine among three potential hackers suspected by the company, whether the third hacker is involed in hacking or not.

Dataset:¶

The Dataset "hack_data.csv" contains 334 attack instances with following attributes:

  1. Session_Connection_Time
  2. Bytes Transferred
  3. Kali_Trace_Used
  4. Servers_Corrupted
  5. Pages_Corrupted
  6. Location
  7. WPM_Typing_Speed
Attribute Description:¶
  1. Session_Connection_Time: It specifies how long the session lasted.
  2. Bytes Transferred: It specifies the Megabytes transferred during session
  3. Kali_Trace_Used : It specifies whether the hacker was using Kali Linux
  4. Servers_Corrupted: It specifies the number of server corrupted during the attack
  5. Pages_Corrupted: It specifies the number of pages illegally accessed
  6. Location: It specifies the location from where attack has come
  7. WPM_Typing_Speed: It specifies the estimated typing speed based on session logs

Location column is discarded because there are chances that hackers would have used VPNs to hide their real locations during the attacks.

All the remaining attributes play a significant role to determine whether third hacker is involved in hacking or not

Note: if attacks are equally divided among two hackers then third hacker is innocent

Prepare the data for Clustering Model¶

For clustering, MLlib library in Spark accepts only the dataframes that have one column. This column should contain all the features in the form of vectors, with each vector corresponding to the features in that particular row.

So, we are using "VectorAssembler" to assemble all the input columns "Session_Connection_Time, Bytes Transferred,Kali_Trace_Used, Servers_Corrupted,Pages_Corrupted,WPM_Typing_Speed" into one outputCol column called "required_cols"

After creating assembler, we transform our data to "assembled_data".

After transformation, our "assembled_data" will contain all the original attributes as well as the newly created attribute, called features.

In [19]:
# Import VectorAssembler
from pyspark.ml.linalg import Vectors
from pyspark.ml.feature import VectorAssembler
In [20]:
# Select the required attributes and store in "required_cols"
required_cols = data.select(['Session_Connection_Time',
                             'Bytes Transferred',
                             'Kali_Trace_Used',
                             'Servers_Corrupted',
                             'Pages_Corrupted',
                             'WPM_Typing_Speed'])
In [21]:
# VectorAssembler is used to assemble all the attributes in the input column(required_cols) into single vector column called 'features'
assembler = VectorAssembler(inputCols = required_cols.columns, outputCol='features')
In [22]:
# After transformation, assembled_data contains original attributes as well as new attribute "feature"
assembled_data = assembler.transform(data)

K-Means Algorithm¶

To determine whether there were two hackers or three hackers involved in hacking, two K-means models are created with one model initialized K = 2 and other model with K = 3.¶

In [28]:
# Build two models with K = 2 and K = 3
kmeans_k2 = KMeans(featuresCol='scaledFeatures', k=2)
kmeans_k3 = KMeans(featuresCol='scaledFeatures', k=3)
In [29]:
# Fit the two models
model_k2 = kmeans_k2.fit(standarized_data)
model_k3= kmeans_k3.fit(standarized_data)

Model Prediction when K = 3¶

When K = 3, the number of instances are not equally divided among three clusters. This goes against the facr that when group of hackers work together, they often divide the attacks equally.

So, there are very less chances that third hacker involved in attack.

Model Prediction when K = 2¶

With K=2, both clusters here have exactly the same number of instances assigned to them. When group of hackers work together, they often divide the attacks equally. Therefore, we conclude that only two hackers were involved with the attack.

So, third hacker is not involved in attack
